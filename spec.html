Request Format

Think of it as a remote function call. Zero or one responses expected.

{% highlight javascript %} { "cmd": "request_type", "params": { ... }, "id": "unique_request_id" } {% endhighlight %}

Success Response

Equivalent to a function result.

{% highlight javascript %} { "return": "response_type", "result": { ... }, "reply_to": "request_id" } {% endhighlight %}

'reply_to' is used to determine which request it corresponds to.

Error Response

Sent back from server when error occurs. Can be sent in addition or instead of the response.

{% highlight javascript %} { "return": "error", "error": { "code": "invalid", "msg": "Error message here" }, "reply_to": "request_id" } {% endhighlight %}

Subscribe To Event

Subscription to 'events'. These can be tick stream, portfolio or any other events.

{% highlight javascript %} { "subscribe": "event_name", "params": { ... } } {% endhighlight %}

Event Response

Happens when the user had subscribed to the event. Server determines how otften it will be called

{% highlight javascript %} { "event": "event_name", "return": { ... } } {% endhighlight %}

Unsubscribe Request

Stop 'events' from server.

{% highlight javascript %} { "unsubscribe": "event_name", "params": { ... } } {% endhighlight %}
